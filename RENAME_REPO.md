# Инструкция по переименованию репозитория

## ⚠️ Важно: Сначала переименуйте на GitHub, затем обновите remote!

## Шаг 1: Переименование на GitHub

1. Перейдите на страницу репозитория: https://github.com/kobyzev-yuri/lse-about.md
2. Нажмите на кнопку **Settings** (в верхней панели репозитория, справа)
3. Прокрутите вниз до секции **Repository name** (в самом низу страницы Settings)
4. Измените название с `lse-about.md` на `lse`
5. Нажмите **Rename**

⚠️ **Внимание**: После переименования GitHub автоматически перенаправит старый URL, но лучше обновить все ссылки.

## Шаг 2: Обновление remote URL в локальном репозитории

**Только после того, как вы переименовали репозиторий на GitHub**, выполните:

```bash
cd /mnt/ai/cnn/lse
git remote set-url origin https://github.com/kobyzev-yuri/lse.git
git remote -v  # Проверьте, что URL обновлен
```

## Проверка

После обновления remote URL проверьте доступность:

```bash
git ls-remote origin
```

Если команда выполняется без ошибок - все настроено правильно.

## Текущий статус

- ✅ Локальный репозиторий настроен на: `https://github.com/kobyzev-yuri/lse-about.md.git`
- ⏳ Ожидается переименование на GitHub: `lse-about.md` → `lse`
- После переименования выполните команду обновления remote URL выше

---

**Примечание**: Если вы получили 404 ошибку, это означает, что репозиторий еще не переименован на GitHub. 
Вернитесь к Шагу 1 и выполните переименование через веб-интерфейс GitHub.
